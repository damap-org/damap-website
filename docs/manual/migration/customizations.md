# Customizations

In v4.x.x, customizations had to be done by technical personnel in the codebase itself.
v5.0.0 moves this responsibility to the tool admins, by offering them user interfaces to make these changes.
This offers several advantages:

- No technical personnel needed to do the customizations
- Faster and centralized customization workflow
- Updates dont break customized elements anymore
- Updates can be applied dynamically without restarting running systems

All new customization options are covered in the [Customization section](../../customization).
During the migration process, admins will need to plan in time to redo their current customizations.
